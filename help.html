Getting Started

The SSH Log Analyzer helps you analyze authentication logs to detect potential brute-force attacks against your SSH servers.

Where to find your SSH logs
  • On most Linux systems: /var/log/auth.log
  • On CentOS/RHEL: /var/log/secure
  • On FreeBSD: /var/log/auth.log
Benefits
  • Enhanced Security

    Identify attack patterns and potential vulnerabilities

  • Efficient Analysis

    Process large log files in seconds

  • Visual Insights

    Visualize attack patterns and trends

  • Actionable Reports

    Generate detailed security reports

Prerequisites
  • SSH Log File

    Access to your server's SSH authentication logs

  • Modern Web Browser

    Chrome, Firefox, Safari, or Edge

  • Basic Knowledge

    Familiarity with SSH and server logs

Step-by-Step Usage Guide

1. Obtain your SSH log file

You'll need access to your server's SSH authentication logs. You can typically find these at /var/log/auth.log on Linux systems. Copy this file to your local machine for analysis.

sudo cp /var/log/auth.log ~/my_auth.log
sudo chmod 644 ~/my_auth.log
2. Upload your log file

On the home page, you can either:

  • Click the upload area to browse for your file
  • Drag and drop your log file directly

The file will be processed locally in your browser - no data is sent to external servers.

Upload SSH Authentication Log File

3. Configure analysis parameters
Required Parameters
Attack Threshold

Minimum number of failed attempts to consider as an attack (default: 5)

Time Window

Time period in minutes during which failed attempts are counted (default: 60)

Optional Parameters
IP Filter

Filter results for a specific IP address

Username Filter

Filter results for a specific username

Date Range

Restrict analysis to a specific time period

4. Analyze the log

Click the "Analyze Log" button to start the analysis. This may take a few moments depending on the size of your log file.

Security Recommendations
For Detected Attacks
  • Implement Rate Limiting

    Limit the number of SSH connection attempts from a single IP

  • Use Key Authentication

    Disable password authentication and use SSH keys

  • Change SSH Port

    Use a non-standard port instead of the default port 22

Additional Measures
  • Install Fail2ban

    Automatically block IPs with repeated failed attempts

  • Review User Accounts

    Remove unused accounts and enforce strong passwords

  • Regular Monitoring

    Schedule regular log analysis and security audits

Troubleshooting

If you're having trouble uploading your log file:

  • Ensure the file is in text format
  • Check that the file has a .log or .txt extension
  • The maximum file size is 100MB
  • Try splitting large log files into smaller chunks

If no attacks are detected in your logs:

  • Try lowering the attack threshold
  • Increase the time window
  • Check that your log file contains SSH authentication events
  • Ensure the log format is compatible with the analyzer

If you encounter issues with report generation:

  • Make sure your browser allows pop-ups for downloading files
  • Try using a different browser
  • Ensure you have sufficient disk space to save the report
  • For very large log files, try filtering by date range

Command-Line Interface

The SSH Log Analyzer is also available as a command-line tool. You can run it directly:

python ssh_log_analyzer.py -f /path/to/auth.log -t 5 -w 60
Available Command Line Options
Option Short Description Default
--file -f Path to the auth.log file /var/log/auth.log
--output -o Output file for the report ssh_attack_report.html
--threshold -t Number of failed attempts to consider as an attack 5
--window -w Time window in minutes to consider consecutive failures 60
--ip -i Filter results for a specific IP address None
--user -u Filter results for a specific username None
--from-date Start date for analysis (YYYY-MM-DD) None
--to-date End date for analysis (YYYY-MM-DD) None
--no-viz Disable visualization generation False
--verbose -v Enable verbose output False
Example Commands
Basic Analysis
python ssh_log_analyzer.py -f /var/log/auth.log
Custom Threshold
python ssh_log_analyzer.py -f /var/log/auth.log -t 3 -w 30
Filter by IP
python ssh_log_analyzer.py -f /var/log/auth.log -i 192.168.1.105
Date Range
python ssh_log_analyzer.py -f /var/log/auth.log --from-date 2023-01-01 --to-date 2023-01-31
